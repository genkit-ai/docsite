This guide shows you how to get started with Genkit in a Node.js app and test it in the Developer UI.

## Prerequisites

Before you begin, make sure your environment meets these requirements:

- Node.js v20 or later
- npm

This guide assumes you're already familiar with building Node.js applications.

## Set up your project

Create a new Node.js project and configure TypeScript:

```sh
mkdir my-genkit-app
cd my-genkit-app
npm init -y

# Set up your source directory
mkdir src
touch src/index.ts

# Install and configure TypeScript
npm install -D typescript tsx
npx tsc --init
```

This initializes a new Node.js project using TypeScript and creates an entry file at `src/index.ts`.

## Install Genkit packages

First, install the Genkit CLI globally. This gives you access to local developer tools, including the Developer UI:

```bash
npm install -g genkit-cli
```

Then, add the following Genkit packages to your project:

```bash
npm install genkit @genkit-ai/googleai
```

- `genkit` provides Genkit core capabilities.
- `@genkit-ai/googleai` provides access to the Google AI Gemini models.

## Configure your model API key

Genkit can work with multiple model providers. This guide uses the **Gemini API**, which offers a free tier and doesn’t require a credit card to get started. If you don't already have a Gemini API key, create one
in Google AI Studio.

## Create your first flow

Now you’re ready to build your first Genkit flow. Flows are functions with built-in type safety, observability, and tooling integration.

Update `src/index.ts` with the following:

```ts
import { googleAI } from '@genkit-ai/googleai';
import { genkit, z } from 'genkit';

// Initialize Genkit with the Google AI plugin
const ai = genkit({
  plugins: [googleAI()],
  model: googleAI.model('gemini-2.5-flash', {
    temperature: 0.8 
  }),
});

// Define input schema
const RecipeInputSchema = z.object({
  ingredient: z.string().describe('Main ingredient or cuisine type'),
  dietaryRestrictions: z.string().optional().describe('Any dietary restrictions'),
});

// Define output schema
const RecipeSchema = z.object({
  title: z.string(),
  description: z.string(),
  prepTime: z.string(),
  cookTime: z.string(),
  servings: z.number(),
  ingredients: z.array(z.string()),
  instructions: z.array(z.string()),
  tips: z.array(z.string()).optional(),
});

// Define a recipe generator flow
export const recipeGeneratorFlow = ai.defineFlow(
  {
    name: 'recipeGeneratorFlow',
    inputSchema: RecipeInputSchema,
    outputSchema: RecipeSchema,
  },
  async (input) => {
    // Create a prompt based on the input
    const prompt = `Create a recipe with the following requirements:
      Main ingredient: ${input.ingredient}
      Dietary restrictions: ${input.dietaryRestrictions || 'none'}`;

    // Generate structured recipe data using the same schema
    const { output } = await ai.generate({
      prompt,
      output: { schema: RecipeSchema },
    });
    
    if (!output) throw new Error('Failed to generate recipe');

    return output;
  }
);

// Run the flow
async function main() {
  const recipe = await recipeGeneratorFlow({ 
    ingredient: 'avocado', 
    dietaryRestrictions: 'vegetarian' 
  });
  
  console.log(recipe);
}

main().catch(console.error);
```

This code sample:

- Defines reusable input and output schemas with [Zod](https://zod.dev/)
- Create a Genkit flow that uses those schemas
- Generates structured recipe data from a prompt and schema
- Runs the flow with a sample input and logs the result

##### Why use flows?

- Type-safe inputs and outputs
- Integrates with the Developer UI
- Easy deployment as APIs
- Built-in tracing and observability

## Test in the Developer UI

The Developer UI is a local tool for inspecting and running Genkit components, like flows, with a visual interface.

### Start the Developer UI

Run the following command from your project directory:

```bash
genkit start -- npx tsx --watch src/index.ts
```

This starts your app, watches for changes, and launches the Developer UI at http://localhost:4000 by default.

:::note
The command after `--` should run the file that defines or imports your Genkit flows. You can use `tsx`, `node`, or other commands based on your setup. See [developer tools](/docs/devtools) for more options.
:::

##### Optional: Add a script to `package.json`

To make starting the Developer UI easier, add this script:

```json
"scripts": {
  "genkit:ui": "genkit start -- npx tsx --watch src/index.ts"
}
```

Then run:

```sh
npm run genkit:ui
```

### Run and debug your flow in the Developer UI

Once the UI is open:

1. Select the `recipeGeneratorFlow` from the list of flows
1. Enter sample input:
   ```json
   {
     "ingredient": "avocado",
     "dietaryRestrictions": "vegetarian"
   }
   ```
1. Click **Run**

The Developer UI will show the structured output of the flow and display a trace of the flow execution steps to help you better understand its behavior and debug issues.
