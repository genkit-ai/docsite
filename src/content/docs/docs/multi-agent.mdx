A powerful application of large language models are LLM-powered agents. An agent
is a system that can carry out complex tasks by planning how to break tasks into
smaller ones, and (with the help of [tool calling](/docs/tool-calling)) execute tasks
that interact with external resources such as databases or even physical
devices.

As you build more tools, you start to run into problems:

- The more tools you add, the more you stretch the model's ability to
  consistently and correctly employ the right tool for the job.
- Some tasks might best be served through a more focused back and forth
  between the user and the agent, rather than by a single tool call.
- Some tasks might benefit from a specialized prompt.

The solution is to create many specialized agents and use a general purpose
orchestrator agent to delegate tasks to them.

## Multi-agent delegation with the `agents()` middleware

The recommended approach is to use `ai.defineAgent()` and the `agents()`
middleware from `@genkit-ai/middleware`:

```typescript
import { agents } from '@genkit-ai/middleware';
import { z } from 'genkit';

// Define specialized sub-agents
const getWebResults = ai.defineTool(
  {
    name: 'getWebResults',
    description: 'Search the web for information on a topic.',
    inputSchema: z.object({ query: z.string() }),
    outputSchema: z.object({ results: z.string() }),
  },
  async (input) => {
    return {
      results: `Top results for "${input.query}": ...`,
    };
  }
);

const researcher = ai.defineAgent({
  name: 'researcher',
  model: 'googleai/gemini-flash-latest',
  system:
    'You are a thorough research assistant. When asked a question, use the getWebResults tool to find information, then provide a clear, well-sourced answer.',
  tools: [getWebResults],
});

const coder = ai.defineAgent({
  name: 'coder',
  model: 'googleai/gemini-flash-latest',
  system:
    'You are an expert programmer. When asked to write code, provide clean, well-commented code with explanations. Use TypeScript by default unless asked otherwise.',
});

// Orchestrator delegates to sub-agents via the `agents` middleware
const orchestratorAgent = ai.defineAgent({
  name: 'orchestrator',
  model: 'googleai/gemini-flash-latest',
  system: `You are a helpful project assistant. You have access to specialized sub-agents:

- **researcher**: Use for research questions, fact-finding, looking up information
- **coder**: Use for writing code, debugging, code explanations

Analyze the user's request and delegate to the appropriate sub-agent using the call_agent tool.
If the request requires both research AND code, call them sequentially.
After receiving sub-agent responses, synthesize a final answer for the user.`,
  use: [agents({ agents: ['researcher', 'coder'] })],
});
```

The `agents()` middleware injects a `call_agent` tool that the orchestrator
model can call. When it does, the middleware runs the appropriate sub-agent
and returns the response as the tool result.

```typescript
const result = await orchestratorAgent.run(
  {
    messages: [{
      role: 'user',
      content: [{ text: 'Research sorting algorithms then write a quicksort in TypeScript.' }],
    }],
  },
  { onChunk: (chunk) => console.log(chunk) }
);
```

For complete documentation on agents including session management, persistence,
interrupts, background execution, and more, see the [Agents](/docs/agents) page.
