Pick the guide that matches how you're building. Each guide is self-contained — you don't need to complete the general quickstart first.

## Start with your app platform

<Lang lang="js">

If you're building a web app or full-stack TypeScript application, start with one of these guides:

- [Next.js App Router](/guides/frameworks/nextjs-app-router) for full-stack React apps using route handlers and the first-party Genkit Next.js integration.
- [Next.js Pages Router](/guides/frameworks/nextjs-pages) if you're adding Genkit to an existing `pages/` app and want the best migration path.
- [SvelteKit](/guides/frameworks/sveltekit) for full-stack Svelte apps using SvelteKit API routes and the Genkit fetch handler.
- [Nuxt](/guides/frameworks/nuxt) for full-stack Vue apps using Nitro server routes and the Genkit fetch handler.
- [Astro](/guides/frameworks/astro) for content-driven sites using Astro SSR with API routes and the Genkit fetch handler.
- [Angular](/guides/frameworks/angular) for Angular SSR and hybrid-rendered applications.
- [Express](/guides/frameworks/express) for existing Node.js backends or custom API servers.
- [Hono](/guides/frameworks/hono) for Web-standard request and response handling across Node.js and edge-style runtimes.
- [Fastify](/guides/frameworks/fastify) for Fastify servers that want to expose Genkit flows through the fetch plugin.
- [Deploy with Firebase](/guides/deployment/firebase) if you want the most integrated Google-first serverless path.
- [Deploy to any platform](/guides/deployment/any-platform) if you're building your own Node.js hosting setup.

If you're not ready to choose a framework-specific path yet, start with the [Quickstart](/docs/get-started), then continue into [Creating flows](/docs/flows) and [Generating content](/docs/models).

</Lang>

<Lang lang="go">

For Go projects, Genkit is a great fit when you want a low-level, model-agnostic SDK behind your own services:

- [Gin](/guides/frameworks/gin) for Gin-based REST APIs and web services.
- [Echo](/guides/frameworks/echo) for Echo servers with built-in middleware support.
- [Chi](/guides/frameworks/chi) for lightweight, composable routers using `net/http`.
- [Deploy with Cloud Run](/guides/deployment/cloud-run) for the most direct managed deployment path.
- [Deploy to any platform](/guides/deployment/any-platform) for custom servers and self-managed environments.

If you're not ready to choose a framework-specific path yet, start with the [Quickstart](/docs/get-started), then continue into [Creating flows](/docs/flows) and [Generating content](/docs/models).

If you're building a Go backend for a separate web or mobile frontend, pair this with the [Web client](/guides/frontend/web-client) or [Flutter](/guides/frontend/flutter) guide.

</Lang>

<Lang lang="dart">

For Dart, Genkit supports both client-heavy and full-stack paths:

- [Flutter](/guides/frontend/flutter) for mobile, desktop, and web apps using the Dart client SDK story.
- [Shelf](/guides/frameworks/shelf) for Dart servers that expose flows and APIs directly.
- [Deploy with Cloud Run](/guides/deployment/cloud-run) for managed backend hosting.
- [Deploy to any platform](/guides/deployment/any-platform) for custom Dart deployments.

If you want to get oriented first, start with the [Quickstart](/docs/get-started) and then explore [Calling Genkit from the client](/docs/client).

</Lang>

<Lang lang="python">

For Python, start from the web framework or deployment model you already use:

- [FastAPI](/guides/frameworks/fastapi) for modern APIs, streaming responses, and typed server applications.
- [Flask](/guides/frameworks/flask) for lightweight Python services and flexible server integrations.
- [Django](/guides/frameworks/django) for adding Genkit flows to Django projects using Django Ninja or REST framework.
- [Deploy with Cloud Run](/guides/deployment/cloud-run) for a managed container path.
- [Deploy to any platform](/guides/deployment/any-platform) for custom Python hosting.

If your Python service is powering a separate web or mobile app, also review the [Web client](/guides/frontend/web-client) or [Flutter](/guides/frontend/flutter) guide.

</Lang>

## Choose by application shape

- **Full-stack single-language app:** Start with the framework guide for your selected language, such as [Next.js App Router](/guides/frameworks/nextjs-app-router), [SvelteKit](/guides/frameworks/sveltekit), [Nuxt](/guides/frameworks/nuxt), [Astro](/guides/frameworks/astro), [Angular](/guides/frameworks/angular), [Flutter](/guides/frontend/flutter), [Shelf](/guides/frameworks/shelf), or [Hono](/guides/frameworks/hono) when you want a lightweight TypeScript server that can target Node.js or fetch-based runtimes.
- **Backend plus separate client app:** Start with the server framework or deployment guide for your backend language, then connect a frontend with the [Web client](/guides/frontend/web-client) or [Flutter](/guides/frontend/flutter) guide.
- **Prototype first, architecture later:** Start with the [Quickstart](/docs/get-started), test your flows in the [Developer tools](/docs/devtools), and then move into a framework guide once the feature takes shape.
