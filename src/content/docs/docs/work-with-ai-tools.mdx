Genkit works well with AI-powered IDEs and coding agents when they can see your local runtime, understand your flows, and look up framework-specific guidance. The fastest way to get there is to use Genkit's AI tooling support and MCP server together.

## Recommended setup

1. Install the [Developer tools](/docs/devtools) so you can run the Genkit CLI locally.
2. Set up the [Genkit MCP server](/docs/mcp-server) in your editor or agent environment.
3. If your language is supported, run the AI tooling bootstrap command to configure project context automatically.

<Lang lang="js go">

For JavaScript and Go projects, Genkit can bootstrap editor and agent setup for you:

- Use [AI-assisted development](/docs/develop-with-ai) to configure supported tools automatically.
- This setup can install the Genkit MCP server, add project instructions, and wire up runtime commands for your AI assistant.

</Lang>

<Lang lang="dart python">

For Dart and Python today, the best path is to configure the [Genkit MCP server](/docs/mcp-server) directly in your editor or AI tool, then run the [Developer tools](/docs/devtools) locally while you work.

</Lang>

## What this unlocks

Once configured, AI tools can help with tasks like:

- discovering flows and other Genkit components in your project
- running flows with test input
- inspecting traces and debugging behavior
- reading Genkit documentation from inside the tool
- staying aligned with the conventions of your selected language
