## API Stability

Genkit follows semantic versioning and provides stability guarantees across different release channels:

- **Stable APIs**: Production-ready with backward compatibility guarantees
- **Beta APIs**: Feature-complete but may have minor breaking changes
- **Alpha APIs**: Experimental features subject to significant changes

<LanguageContent lang="js">
  Learn more about [API stability channels](/js/api-stability) for JavaScript.
</LanguageContent>

## Contributing to Documentation

The API documentation is generated from source code comments and type definitions. To contribute:

<LanguageContent lang="js">
  1. Update JSDoc comments in the source code 2. Ensure TypeScript types are properly documented 3. Submit a pull
  request to the [Genkit repository](https://github.com/firebase/genkit)
</LanguageContent>

<LanguageContent lang="go">
  1. Update Go doc comments in the source code 2. Ensure public APIs have comprehensive documentation 3. Submit a pull
  request to the [Genkit repository](https://github.com/firebase/genkit)
</LanguageContent>

<LanguageContent lang="python">
  1. Update docstrings following Google style guidelines 2. Ensure type hints are properly documented 3. Submit a pull
  request to the [Genkit repository](https://github.com/firebase/genkit)
</LanguageContent>
