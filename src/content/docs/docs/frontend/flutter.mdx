Use this guide when you have a Genkit backend exposing flows as HTTP endpoints and want to call them from a Flutter app.

This is the recommended architecture for production apps: your model API keys stay on the server, and Flutter calls the backend over HTTP. Your backend can be written in any language — Node.js, Go, Python, or Dart.

## Before you begin

- Flutter SDK
- Dart SDK 3.10.0 or later
- A Genkit backend with at least one flow exposed as an HTTP endpoint. See [Get started](/docs/get-started) to set one up.

## Create a project

Create a new Flutter app if you don't already have one:

```bash
flutter create my_genkit_flutter
cd my_genkit_flutter
```

## Project layout

<FileTree>

- lib
  - **main.dart**
- pubspec.yaml

</FileTree>

## Install Genkit packages

Add the Genkit Dart client package:

```bash
flutter pub add genkit
```

## Configure your backend

Start your Genkit backend and make sure it exposes the recipe flow as an HTTP endpoint. The URL depends on which framework and port your backend uses — for example:

```text
http://localhost:8080/recipeGeneratorFlow
```

See [Get started](/docs/get-started) if you need to set up a backend first.

## Call the recipe flow from Flutter

Replace `lib/main.dart` with:

```dart title="lib/main.dart"
import 'package:flutter/material.dart';
import 'package:genkit/client.dart';

// Replace this URL with the address of your Genkit backend
final recipeGeneratorFlow = defineRemoteAction<Map<String, dynamic>>(
  url: 'http://localhost:8080/recipeGeneratorFlow',
  fromResponse: (data) => Map<String, dynamic>.from(data),
);

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return const MaterialApp(home: RecipePage());
  }
}

class RecipePage extends StatefulWidget {
  const RecipePage({super.key});

  @override
  State<RecipePage> createState() => _RecipePageState();
}

class _RecipePageState extends State<RecipePage> {
  final ingredientController = TextEditingController(text: 'avocado');
  final dietaryRestrictionsController =
      TextEditingController(text: 'vegetarian');
  Map<String, dynamic>? recipe;
  bool isLoading = false;

  Future<void> generateRecipe() async {
    setState(() => isLoading = true);

    final result = await recipeGeneratorFlow(input: {
      'ingredient': ingredientController.text,
      'dietaryRestrictions': dietaryRestrictionsController.text,
    });

    setState(() {
      recipe = result;
      isLoading = false;
    });
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Recipe generator')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: Column(
          children: [
            TextField(
              controller: ingredientController,
              decoration: const InputDecoration(labelText: 'Ingredient'),
            ),
            TextField(
              controller: dietaryRestrictionsController,
              decoration: const InputDecoration(labelText: 'Dietary restrictions'),
            ),
            const SizedBox(height: 16),
            FilledButton(
              onPressed: isLoading ? null : generateRecipe,
              child: const Text('Generate recipe'),
            ),
            const SizedBox(height: 16),
            if (recipe != null) Expanded(child: SingleChildScrollView(child: Text(recipe.toString()))),
          ],
        ),
      ),
    );
  }
}
```

## Run your application

Start your backend first, then run Flutter:

```bash
flutter run
```

If you're using Flutter web during development, `genkit start:flutter` is also a convenient option:

```bash
genkit start:flutter -- -d chrome
```

## Test the full flow

Once your backend and Flutter app are both running:

- enter an ingredient and optional dietary restriction
- tap `Generate recipe`
- confirm the structured recipe appears in the app

## Test in the Developer UI

The **Developer UI** is a local tool for testing and inspecting Genkit components, like flows, with a visual interface.

### Start the Developer UI

The Genkit CLI is required to run the Developer UI. If you followed the installation steps above, you already have it installed.

To inspect your flows in the Developer UI, run your backend with the Genkit CLI. For example, if your backend is a Dart server:

```bash
genkit start -- dart run
```

The command will print the Dev UI URL:

```
Genkit Developer UI: http://localhost:4000
```

### Run and inspect flows

In the Developer UI:

1. Select `recipeGeneratorFlow` from the list of flows.
2. Enter sample input:

   ```json
   {
     "ingredient": "avocado",
     "dietaryRestrictions": "vegetarian"
   }
   ```

3. Click **Run**

   You'll see the generated recipe as structured output, along with a visual trace of the AI generation process for debugging and optimization.
