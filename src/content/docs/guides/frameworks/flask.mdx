Use this guide when you're starting from a Flask app and want to add a Genkit flow without changing your existing Python service architecture.

## Before you begin

- Python 3.10 or later
- uv package manager

This guide assumes you're already familiar with building Flask applications.

## Create a project

Create a new project with uv:

```bash
# Create and navigate to a new directory
mkdir my-genkit-flask
cd my-genkit-flask

# Initialize the project with uv
uv init --no-readme --python 3.10
```

## Project layout

<FileTree>

- pyproject.toml
- **main.py**

</FileTree>

## Install Genkit packages

Install the Genkit CLI:

```bash
curl -sL cli.genkit.dev | bash
```

Then install the packages you need in your project:

```bash
uv add flask genkit genkit-plugin-google-genai genkit-plugin-flask
```

## Configure your model API key

This guide uses the Gemini API from Google AI Studio.

```bash
export GEMINI_API_KEY=<your API key>
```

## Create your application

Create `main.py` with the same recipe flow used in the main quickstart, then expose it through Flask:

```python title="main.py"
from typing import Optional

from flask import Flask
from pydantic import BaseModel, Field

from genkit import Genkit
from genkit.plugins.flask import genkit_flask_handler
from genkit.plugins.google_genai import GoogleAI

ai = Genkit(
    plugins=[GoogleAI()],
    model='googleai/gemini-2.5-flash',
)

app = Flask(__name__)


class RecipeInput(BaseModel):
    ingredient: str = Field(description='Main ingredient or cuisine type')
    dietary_restrictions: Optional[str] = Field(
        default=None,
        description='Any dietary restrictions',
    )


class Recipe(BaseModel):
    title: str
    description: str
    prep_time: str
    cook_time: str
    servings: int
    ingredients: list[str]
    instructions: list[str]
    tips: list[str] | None = None


@app.post('/recipeGeneratorFlow')
@genkit_flask_handler(ai)
@ai.flow()
async def recipe_generator_flow(input_data: RecipeInput) -> Recipe:
    dietary_restrictions = input_data.dietary_restrictions or 'none'

    prompt = f"""Create a recipe with the following requirements:
        Main ingredient: {input_data.ingredient}
        Dietary restrictions: {dietary_restrictions}"""

    result = await ai.generate(
        prompt=prompt,
        output_schema=Recipe,
    )

    if not result.output:
        raise ValueError('Failed to generate recipe')

    return result.output


if __name__ == '__main__':
    app.run(debug=True)
```

## Run your application

Run your app locally:

```bash
uv run flask --app main.py run
```

You can also launch it through the Developer UI:

```bash
genkit start -- uv run flask --app main.py run
```

## Test your flow

With the server running, call your flow over HTTP:

```bash
curl -X POST http://127.0.0.1:5000/recipeGeneratorFlow \
  -H "Content-Type: application/json" \
  -d '{"data":{"ingredient":"avocado","dietary_restrictions":"vegetarian"}}'
```

You should receive a structured recipe in the response.

## Test in the Developer UI

The **Developer UI** is a local tool for testing and inspecting Genkit components, like flows, with a visual interface.

### Start the Developer UI

The Genkit CLI is required to run the Developer UI. If you followed the installation steps above, you already have it installed.

To inspect your app with Genkit Dev UI, run:

```bash
genkit start -- uv run flask --app main.py run
```

The command will print the Dev UI URL:

```
Genkit Developer UI: http://localhost:4000
```

### Run and inspect flows

In the Developer UI:

1. Select `recipe_generator_flow` from the list of flows.
2. Enter sample input:

   ```json
   {
     "ingredient": "avocado",
     "dietary_restrictions": "vegetarian"
   }
   ```

3. Click **Run**

   You'll see the generated recipe as structured output, along with a visual trace of the AI generation process for debugging and optimization.
