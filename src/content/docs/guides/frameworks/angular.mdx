Use this guide when you're starting from an Angular app and want a practical Genkit setup with a server route and frontend call.

## Before you begin

- Node.js v20 or later
- npm
- Angular CLI

This guide assumes you're already familiar with building Angular applications.

## Create a project

This guide uses Angular with SSR and server routing:

```bash
ng new --ssr --server-routing
```

## Project layout

<FileTree>

- package.json
- src
  - app
    - **app.component.html**
    - **app.component.ts**
  - genkit
    - **recipeGeneratorFlow.ts**
  - **server.ts**

</FileTree>

## Install Genkit packages

Install the Genkit CLI globally:

```bash
npm install -g genkit-cli
```

Then install the packages you need in your project:

```bash
npm install genkit @genkit-ai/google-genai @genkit-ai/express
npm install -D tsx
```

## Configure your model API key

This guide uses the Gemini API from Google AI Studio.

```bash
export GEMINI_API_KEY=<your API key>
```

## Define your recipe flow

Create `src/genkit/recipeGeneratorFlow.ts`:

```ts title="src/genkit/recipeGeneratorFlow.ts"
import { googleAI } from '@genkit-ai/google-genai';
import { genkit, z } from 'genkit';

const ai = genkit({
  plugins: [googleAI()],
  model: googleAI.model('gemini-2.5-flash', {
    temperature: 0.8,
  }),
});

const RecipeInputSchema = z.object({
  ingredient: z.string().describe('Main ingredient or cuisine type'),
  dietaryRestrictions: z
    .string()
    .optional()
    .describe('Any dietary restrictions'),
});

const RecipeSchema = z.object({
  title: z.string(),
  description: z.string(),
  prepTime: z.string(),
  cookTime: z.string(),
  servings: z.number(),
  ingredients: z.array(z.string()),
  instructions: z.array(z.string()),
  tips: z.array(z.string()).optional(),
});

export const recipeGeneratorFlow = ai.defineFlow(
  {
    name: 'recipeGeneratorFlow',
    inputSchema: RecipeInputSchema,
    outputSchema: RecipeSchema,
  },
  async (input) => {
    const prompt = `Create a recipe with the following requirements:
      Main ingredient: ${input.ingredient}
      Dietary restrictions: ${input.dietaryRestrictions || 'none'}`;

    const { output } = await ai.generate({
      prompt,
      output: { schema: RecipeSchema },
    });

    if (!output) throw new Error('Failed to generate recipe');
    return output;
  },
);
```

## Add a server route

In `src/server.ts`, add:

```ts title="src/server.ts"
import { expressHandler } from '@genkit-ai/express';
import { recipeGeneratorFlow } from './genkit/recipeGeneratorFlow';
```

Then register the route after your `app` is created:

```ts
app.use(express.json());
app.post('/api/recipeGeneratorFlow', expressHandler(recipeGeneratorFlow));
```

## Call your flow from Angular

Update `src/app/app.component.ts`:

```ts title="src/app/app.component.ts"
import { Component, signal } from '@angular/core';
import { FormsModule } from '@angular/forms';
import { runFlow } from 'genkit/beta/client';

@Component({
  selector: 'app-root',
  imports: [FormsModule],
  templateUrl: './app.component.html',
})
export class AppComponent {
  ingredient = signal('avocado');
  dietaryRestrictions = signal('vegetarian');
  recipe = signal<any | null>(null);

  async generateRecipe() {
    const result = await runFlow({
      url: '/api/recipeGeneratorFlow',
      input: {
        ingredient: this.ingredient(),
        dietaryRestrictions: this.dietaryRestrictions(),
      },
    });

    this.recipe.set(result);
  }
}
```

Update `src/app/app.component.html`:

```html title="src/app/app.component.html"
<main>
  <h1>Recipe generator</h1>
  <input
    [ngModel]="ingredient()"
    (ngModelChange)="ingredient.set($event)"
    placeholder="Ingredient"
  />
  <input
    [ngModel]="dietaryRestrictions()"
    (ngModelChange)="dietaryRestrictions.set($event)"
    placeholder="Dietary restrictions"
  />
  <button (click)="generateRecipe()">Generate recipe</button>

  @if (recipe()) {
    <pre>{{ recipe() | json }}</pre>
  }
</main>
```

## Run your application

Run your app locally as normal:

```bash
npm start
```

You can also launch the Developer UI alongside your flow code:

```bash
genkit start -- npx tsx --watch src/genkit/recipeGeneratorFlow.ts
```

## Test your flow

You can test the route directly:

```bash
curl -X POST http://localhost:4200/api/recipeGeneratorFlow \
  -H "Content-Type: application/json" \
  -d '{"data":{"ingredient":"avocado","dietaryRestrictions":"vegetarian"}}'
```

## Test in the Developer UI

The **Developer UI** is a local tool for testing and inspecting Genkit components, like flows, with a visual interface.

### Start the Developer UI

The Genkit CLI is required to run the Developer UI. If you followed the installation steps above, you already have it installed.

Run the following command from your project root:

```bash
genkit start -- npx tsx --watch src/genkit/recipeGeneratorFlow.ts
```

This starts your app and launches the Developer UI at `http://localhost:4000` by default.

:::note
The command after `--` should run the file that defines or imports your Genkit components. You can use `tsx`, `node`, or other commands based on your setup. Learn more in [developer tools](/docs/devtools).
:::

##### Optional: Add an npm script

To make starting the Developer UI easier, add the following to your `package.json` scripts:

```json
"scripts": {
  "genkit:ui": "genkit start -- npx tsx --watch src/genkit/recipeGeneratorFlow.ts"
}
```

Then run it with:

```bash
npm run genkit:ui
```

### Run and inspect flows

In the Developer UI:

1. Select `recipeGeneratorFlow` from the list of flows.
2. Enter sample input:

   ```json
   {
     "ingredient": "avocado",
     "dietaryRestrictions": "vegetarian"
   }
   ```

3. Click **Run**
