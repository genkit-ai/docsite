Use this page to choose the best Genkit integration path for your Next.js app.

For complete API reference documentation, see the [Genkit Next.js Plugin API Reference](https://js.api.genkit.dev/modules/_genkit-ai_next.html).

## Choose your router

- [Next.js App Router](/guides/frameworks/nextjs-app-router) is the primary first-party integration path. Use this if you're starting a new app or already using `app/` route handlers.
- [Next.js Pages Router](/guides/frameworks/nextjs-pages) is the better fit if your app still uses `pages/` and you want guidance on adding Genkit without migrating everything at once.

## Recommendation

If you're choosing today, start with [Next.js App Router](/guides/frameworks/nextjs-app-router). That's the route Genkit JS supports directly through `@genkit-ai/next`.

If you're already on Pages Router, keep your current UI architecture and expose Genkit through a small companion server such as [Express](/guides/frameworks/express) or [Hono](/guides/frameworks/hono), then migrate endpoints later if you want tighter Next.js integration.
