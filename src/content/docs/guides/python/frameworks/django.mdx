Use this guide when you're starting from a Django project and want to add a Genkit flow as an HTTP endpoint.

Two approaches are covered below. **Django Ninja** is recommended for most projects — it's async-first and Pydantic-native, which aligns naturally with Genkit flows. The **async view** approach is available if you want to stay closer to Django's built-in primitives or need fine-grained control over streaming with SSE.

## Before you begin

- Python 3.10 or later
- uv package manager

This guide assumes you're already familiar with building Django applications.

## Create a project

Create a new project with uv:

```bash
mkdir my-genkit-django
cd my-genkit-django
uv init --no-readme --python 3.10
```

## Bootstrap your Django project

If you don't already have a Django project:

```bash
uv run django-admin startproject myproject .
uv run python manage.py startapp recipes
```

<TabItem label="Django Ninja (recommended)">

[Django Ninja](https://django-ninja.dev) is an async-first API framework built on top of Django. It uses Pydantic for request and response validation, which aligns naturally with how Genkit flows are typed, and generates OpenAPI docs automatically.

#### Install packages

```bash
uv add django django-ninja django-cors-headers uvicorn genkit genkit-plugin-google-genai
```

#### Create your view

```python title="recipes/views.py"
from typing import Optional

from ninja import NinjaAPI, Schema
from pydantic import Field

from genkit import Genkit
from genkit.plugins.google_genai import GoogleAI

ai = Genkit(
    plugins=[GoogleAI()],
    model='googleai/gemini-2.5-flash',
)

api = NinjaAPI()

class RecipeInput(Schema):
    ingredient: str = Field(description='Main ingredient or cuisine type')
    dietary_restrictions: Optional[str] = Field(
        default=None,
        description='Any dietary restrictions',
    )

class Recipe(Schema):
    title: str
    description: str
    prep_time: str
    cook_time: str
    servings: int
    ingredients: list[str]
    instructions: list[str]
    tips: list[str] | None = None

@ai.flow()
async def recipe_generator_flow(input_data: RecipeInput) -> Recipe:
    dietary_restrictions = input_data.dietary_restrictions or 'none'

    prompt = f"""Create a recipe with the following requirements:
        Main ingredient: {input_data.ingredient}
        Dietary restrictions: {dietary_restrictions}"""

    result = await ai.generate(
        prompt=prompt,
        output_schema=Recipe,
    )

    if not result.output:
        raise ValueError('Failed to generate recipe')

    return result.output

@api.post('/recipeGeneratorFlow')
async def recipe_generator_endpoint(request, data: RecipeInput):
    result = await recipe_generator_flow(data)
    return {'result': result.model_dump()}
```

#### Wire up the URL

```python title="myproject/urls.py"

    path('', api.urls),
]
```

<TabItem label="Async views with SSE">

Use this approach to stay closer to Django's built-in primitives, or when you need fine-grained control over streaming responses using [Server-Sent Events](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events).

#### Install packages

```bash
uv add django django-cors-headers uvicorn genkit genkit-plugin-google-genai
```

#### Create your view

```python title="recipes/views.py"
import json
from typing import Optional

from django.http import JsonResponse, StreamingHttpResponse
from django.views import View
from pydantic import BaseModel, Field

from genkit import Genkit
from genkit.plugins.google_genai import GoogleAI

ai = Genkit(
    plugins=[GoogleAI()],
    model='googleai/gemini-2.5-flash',
)

class RecipeInput(BaseModel):
    ingredient: str = Field(description='Main ingredient or cuisine type')
    dietary_restrictions: Optional[str] = Field(
        default=None,
        description='Any dietary restrictions',
    )

class Recipe(BaseModel):
    title: str
    description: str
    prep_time: str
    cook_time: str
    servings: int
    ingredients: list[str]
    instructions: list[str]
    tips: list[str] | None = None

@ai.flow()
async def recipe_generator_flow(input_data: RecipeInput) -> Recipe:
    dietary_restrictions = input_data.dietary_restrictions or 'none'

    prompt = f"""Create a recipe with the following requirements:
        Main ingredient: {input_data.ingredient}
        Dietary restrictions: {dietary_restrictions}"""

    result = await ai.generate(
        prompt=prompt,
        output_schema=Recipe,
    )

    if not result.output:
        raise ValueError('Failed to generate recipe')

    return result.output

class RecipeGeneratorFlowView(View):
    async def post(self, request) -> JsonResponse | StreamingHttpResponse:
        body = json.loads(request.body)
        data = body.get('data')
        if data is None:
            return JsonResponse(
                {'error': 'request body must be wrapped in {"data": ...}'},
                status=400,
            )

        input_data = RecipeInput(**data)

        if request.headers.get('Accept') == 'text/event-stream':
            async def event_stream():
                stream_response = recipe_generator_flow.stream(input_data)
                async for chunk in stream_response.stream:
                    yield f'data: {json.dumps({"message": chunk})}\n\n'
                result = await stream_response.response
                yield f'data: {json.dumps({"result": result.model_dump()})}\n\n'

            return StreamingHttpResponse(event_stream(), content_type='text/event-stream')

        result = await recipe_generator_flow(input_data)
        return JsonResponse({'result': result.model_dump()})
```

#### Wire up the URL

```python title="myproject/urls.py"
from django.urls import path
from recipes.views import RecipeGeneratorFlowView

urlpatterns = [
    path('recipeGeneratorFlow', RecipeGeneratorFlowView.as_view()),
]
```

## Run your application

Both options use Uvicorn to run Django with ASGI support:

```bash
uv run uvicorn myproject.asgi:application --reload
```

You can also launch through the Developer UI:

```bash
genkit start -- uv run uvicorn myproject.asgi:application --reload
```

## Test your flow

With the server running, call your flow over HTTP:

```bash
curl -X POST http://localhost:8000/recipeGeneratorFlow \
  -H "Content-Type: application/json" \
  -d '{"data":{"ingredient":"avocado","dietary_restrictions":"vegetarian"}}'
```

You should receive a structured recipe in the response.

## Test in the Developer UI

The **Developer UI** is a local tool for testing and inspecting Genkit components, like flows, with a visual interface.

To inspect your app with Genkit Dev UI, run:

```bash
genkit start -- uv run uvicorn myproject.asgi:application --reload
```

The command will print the Dev UI URL:

```
Genkit Developer UI: http://localhost:4000
```

### Run and inspect flows

In the Developer UI:

1. Select `recipe_generator_flow` from the list of flows.
2. Enter sample input:

   ```json
   {
     "ingredient": "avocado",
     "dietary_restrictions": "vegetarian"
   }
   ```

3. Click **Run**

   You'll see the generated recipe as structured output, along with a visual trace of the AI generation process for debugging and optimization.
