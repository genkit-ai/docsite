Use this guide when you're building with Hono and want to expose Genkit flows over HTTP using `@genkit-ai/fetch`. This pattern works well for Node.js and other fetch-based runtimes.

## Before you begin

- Node.js v20 or later
- npm

This guide assumes you're already familiar with building TypeScript server applications.

## Create a project

If you don't already have a Hono project, create one:

```bash
# Create a new Hono app with the Node.js template
npm create hono@latest my-genkit-hono -- --template nodejs --pm npm --install

# Navigate to the project
cd my-genkit-hono
```

</FileTree>

## Install Genkit packages

Install the Genkit CLI globally:

```bash
npm install -g genkit-cli
```

Then install the packages you need in your project:

```bash
npm install genkit @genkit-ai/google-genai @genkit-ai/fetch hono
npm install -D @hono/node-server
```

## Configure your model API key

This guide uses the Gemini API from Google AI Studio.

```bash
export GEMINI_API_KEY=<your API key>
```

## Create your application

Update `src/index.ts` with the same recipe flow used in the main quickstart, then expose it through Hono:

```ts title="src/index.ts"
import { serve } from '@hono/node-server';
import { handleFlows } from '@genkit-ai/fetch';
import { googleAI } from '@genkit-ai/google-genai';
import { genkit, z } from 'genkit';
import { Hono } from 'hono';

const ai = genkit({
  plugins: [googleAI()],
  model: googleAI.model('gemini-2.5-flash', {
    temperature: 0.8,
  }),
});

const RecipeInputSchema = z.object({
  ingredient: z.string().describe('Main ingredient or cuisine type'),
  dietaryRestrictions: z
    .string()
    .optional()
    .describe('Any dietary restrictions'),
});

const RecipeSchema = z.object({
  title: z.string(),
  description: z.string(),
  prepTime: z.string(),
  cookTime: z.string(),
  servings: z.number(),
  ingredients: z.array(z.string()),
  instructions: z.array(z.string()),
  tips: z.array(z.string()).optional(),
});

const recipeGeneratorFlow = ai.defineFlow(
  {
    name: 'recipeGeneratorFlow',
    inputSchema: RecipeInputSchema,
    outputSchema: RecipeSchema,
  },
  async (input) => {
    const prompt = `Create a recipe with the following requirements:
      Main ingredient: ${input.ingredient}
      Dietary restrictions: ${input.dietaryRestrictions || 'none'}`;

    const { output } = await ai.generate({
      prompt,
      output: { schema: RecipeSchema },
    });

    if (!output) throw new Error('Failed to generate recipe');
    return output;
  },
);

const app = new Hono();
const flows = [recipeGeneratorFlow];

app.post('/api/:flowName', async (c) => {
  return handleFlows(c.req.raw, flows, '/api/');
});

serve(
  {
    fetch: app.fetch,
    port: 3780,
  },
  (info) => {
    console.log(`Hono server listening on http://localhost:${info.port}`);
  },
);
```

## Run your application

Run your app locally:

```bash
npx tsx src/index.ts
```

You can also launch it through the Developer UI:

```bash
genkit start -- npx tsx --watch src/index.ts
```

## Test your flow

With the server running, call your flow over HTTP:

```bash
curl -X POST http://localhost:3780/api/recipeGeneratorFlow \
  -H "Content-Type: application/json" \
  -d '{"data":{"ingredient":"avocado","dietaryRestrictions":"vegetarian"}}'
```

You should receive a structured recipe in the response.

## Test in the Developer UI

The **Developer UI** is a local tool for testing and inspecting Genkit components, like flows, with a visual interface.

### Start the Developer UI

The Genkit CLI is required to run the Developer UI. If you followed the installation steps above, you already have it installed.

Run the following command from your project root:

```bash
genkit start -- npx tsx --watch src/index.ts
```

This starts your app and launches the Developer UI at `http://localhost:4000` by default.

:::note
The command after `--` should run the file that defines or imports your Genkit components. You can use `tsx`, `node`, or other commands based on your setup. Learn more in [developer tools](/docs/js/devtools/).
:::

##### Optional: Add an npm script

To make starting the Developer UI easier, add the following to your `package.json` scripts:

```json
"scripts": {
  "genkit:ui": "genkit start -- npx tsx --watch src/index.ts"
}
```

Then run it with:

```bash
npm run genkit:ui
```

### Run and inspect flows

In the Developer UI:

1. Select `recipeGeneratorFlow` from the list of flows.
2. Enter sample input:

   ```json
   {
     "ingredient": "avocado",

   }
   ```

3. Click **Run**
